# 产品需求文档 (PRD) - 西电宁波校友新年家宴抽奖系统

## 1. 项目简介
本项目旨在为西电宁波校友会新年家宴提供一个简单、稳定且具有视觉冲击力的现场抽奖系统。系统需在大屏幕上运行，通过动态效果和流畅的交互，烘托现场节日气氛。

## 2. 功能需求

### 2.1 核心抽奖逻辑
- **名单导入**: 支持预设参与抽奖的校友名单 (目前通过代码配置)。
- **随机抽取**: 点击开始后，系统需通过随机算法从剩余未中奖名单中选取幸运儿。
- **去重机制**: 
  - 已中奖人员自动从后续奖池中剔除，确保每人只有一次中奖机会。
  - 支持单轮抽取单个或多个名额（视具体逻辑而定，当前版本支持单人抽取）。
- **奖项管理**:
  - 支持多轮抽奖（三等奖 -> 马不停蹄奖 -> 二等奖 -> ... -> 特等奖）。
  - 支持切换上一轮/下一轮奖项。
  - 自动判断当前奖项是否已抽完。

### 2.2 用户界面 (UI) 与 交互 (UX)
- **视觉风格**: 深色背景+霓虹光效，配合烟花和飘落元素，营造高端喜庆的年会氛围。
- **主屏幕信息**:
  - 活动标题：“向阳·向蔚来 - 1.31 西电宁波校友新年家宴”。
  - 当前奖项信息：奖品名称、奖品描述、剩余名额。
  - 滚动显示区域：用于展示正在快速滚动的名字。
  - 控制按钮：开始/停止抽奖。
- **动画效果**:
  - 名字滚动动画。
  - 中奖时刻的烟花庆祝特效。
  - 按钮的辉光呼吸效果。
- **辅助信息栏**:
  - **侧边栏**: 展示所有奖项列表及当前完成进度。
  - **底部/侧边**: 展示历史中奖名单。

### 2.3 异常处理
- 奖池抽空提示：当某奖项抽完时，禁用抽奖按钮并提示切换。
- 全员中奖提示：当所有参与者都已中奖时，给予相应提示。

## 3. 非功能需求
- **兼容性**: 需适配主流浏览器 (Chrome/Edge)，推荐全屏运行 (F11)。
- **性能**: 动画流畅，长时间运行不卡顿。
- **部署**: 纯静态页面，可直接本地打开或通过简单的 HTTP Server 运行，无需依懒复杂后端。
- **开源与发布**:
  - 代码仓库需包含标准的 `.gitignore`、`README.md` 和 `LICENSE` 文件。
  - 需剔除所有敏感数据（如个人隐私信息）和大文件（如原始素材）后再发布。

## 4. 数据字典
- **Attendants (参与者)**: 姓名列表 (String Array).
- **PrizeConfig (奖项配置)**:
  - Name (名称)
  - Desc (描述)
  - Count (数量)
  - Icon (图标)
